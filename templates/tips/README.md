# PairFlow Tip 模板

此目录包含 PairFlow 所有 MCP 工具返回的 `tip` 字段模板。fork 维护者可直接编辑这些文件来定制 AI 行动指引，无需修改 TypeScript 源码。

## 模板语法

- 使用 `{{variable_name}}` 双花括号占位符
- 每个模板文件包含 `[行动]`（必需）、`[产出]`（可选）、`[当前]`（可选）三段
- 段标记只能按 `[行动]` → `[产出]` → `[当前]` 顺序出现
- 渲染后由 `formatTip()` 统一组装并去重空段

## 允许的变量

每个模板只可使用其对应模板键声明的变量。变量列表以 `src/tip-template.ts` 中 `TEMPLATE_SPECS` 的 `allowed` 字段为准。启动时会校验——模板引用了未声明的变量会导致启动失败。

## 修改后生效

修改模板后重启 PairFlow Server 生效。模板在启动时一次性加载到内存并缓存。

## 严格失败

模板文件缺失/损坏、引用了未声明的变量、缺少必需变量均会导致 Server 启动失败，错误信息会包含模板键、文件路径和具体原因。不存在静默回退到代码默认值的行为。

## 示例：编辑需求分析第一轮模板

编辑 `requirements/r1.md`：

```markdown
[行动]
请先阅读任务文档 {{task_path}}，然后按以下维度做深入的需求分析：
（你的自定义文案）

[产出]
完成后 git commit，调用 submit，file_path = {{file_path}}

[当前]
你是 {{identity_label}}。当前第 {{round}} 轮{{phase_label}}。
```

保存后重启 PairFlow 即可生效。

## 注意事项

- 不要在模板中存放 token、密钥等敏感信息
- 模板文件编码必须为 UTF-8
- `{{variable}}` 中的变量名区分大小写，仅允许字母、数字和下划线
- 不要使用 `{{action}}`、`{{product}}`、`{{current}}` 等整句逃生变量——模板应持有完整可编辑句子
