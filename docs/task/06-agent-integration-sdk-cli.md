# Agent 集成层：官方 SDK 与短生命周期 CLI

**状态**：待处理

PairFlow 当前主要通过 MCP 工具直接暴露协作协议。不同 Agent 需要各自处理会话凭证、长轮询重试、结构化响应和下一步动作，容易出现重复实现与行为不一致。需要提供官方 Agent 集成层，降低 Codex、Claude 等客户端接入 PairFlow 的成本。

**设计规格**：`docs/design.md`

## 已确认方案

采用“官方 SDK + 短生命周期 CLI”方案：

1. PairFlow 提供官方客户端 SDK，集中封装 HTTP/MCP 调用、认证、响应解析与协议错误。（用户、codex）
2. 提供基于该 SDK 的 CLI，作为 Codex 等可执行本地命令的 Agent 的首选接入方式。（用户、codex）
3. 每条 CLI 命令按需启动、完成一次操作后退出；不要求启动常驻客户端或本地代理进程。（用户、codex）
4. PairFlow Server 仍是唯一需要持续运行的服务。（用户、codex）
5. 会话 token 保存在仓库外的用户级会话存储中，后续命令按 workflow 与 identity 自动取用；不得写入项目文件或提交到 Git。（用户、codex）
6. CLI 负责把结构化响应转成 Agent 可直接执行的明确结果，包括下一动作、原因和当前协作状态。（codex）
7. `wait` 的单次请求超时、取消和 latest-wins 重试由 SDK/CLI 封装，避免每个 Agent 重复实现。（codex）
8. 协议演进与契约治理并入本任务，不再单独立项；SDK/CLI 统一处理服务版本、能力声明、兼容性判断与不兼容错误。（用户、codex）

## 目标使用方式

Agent 通过 CLI 完成以下协作操作，具体命令名和参数在实施设计中确定：

- 建立或恢复参与者会话；
- 确认任务；
- 等待行动权；
- 提交本轮产出；
- 推进阶段或结束工作流；
- 查询当前协作状态；
- 清理本地会话凭证。

SDK 是 CLI 的协议实现基础，也允许具备原生集成能力的 Agent 直接调用。CLI 不负责 Agent 的推理、代码修改、Git 操作或双方结论判断。

**永久边界：PairFlow 的 Server、SDK、CLI 及任何后续组件都不得执行 Git 命令。** Git 操作始终由 Agent 或用户在 PairFlow 之外完成；PairFlow 只接收调用方主动提供的 commit hash 等声明数据。

## 范围

### 必须覆盖（P0）

- SDK 对现有 PairFlow 协作接口的类型安全封装。
- workflow、identity 与 token 的用户级会话管理。
- CLI 对完整主流程的命令覆盖。
- 长轮询超时后的透明续等、取消处理和 latest-wins 语义。
- 结构化成功、等待、告警和错误响应的稳定输出。
- Server 协议版本与能力信息的读取、类型定义和兼容性检查。
- 客户端与 Server 不兼容时的快速失败，以及包含升级方向的明确诊断。
- 面向 Agent 的机器可读输出，以及面向用户诊断的清晰错误信息。
- Windows 环境下可直接使用，并具备自动化测试和最小使用文档。

### 本任务不做

- 不启动常驻客户端、守护进程或 MCP 代理。
- 不通过 SDK 判断 Agent 是否正在推理、编码或执行测试。
- 不增加 heartbeat、active、blocked 等持续活性状态。
- 不执行任何 Git 命令，包括但不限于状态检查、暂存、提交、分支读取、历史查询、rebase、squash、push 或 PR 操作。
- 不改变 PairFlow 的阶段、轮次、行动权和评审规则。
- 不把 token 保存到仓库目录、任务文档或 handoff 产物。

## 质量与安全约束

- token 不得出现在默认日志、标准输出或异常堆栈中。
- 用户级会话存储应使用最小必要权限，并支持显式清理和失效恢复。
- SDK 与 CLI 对同一服务响应必须具有一致语义，不得各自维护一套协议判断。
- 协议兼容规则必须集中定义；新增可选能力不得迫使旧客户端错误地执行未知动作。
- 兼容性检查不得依赖解析面向人的 tip 文案。
- CLI 的自动重试只适用于可安全重试的等待类操作，不得自动重复提交或推进状态。
- CLI 退出不影响 Server 中已经完成的 claim、submit 或 advance 状态变更。
- SDK/CLI 不得通过 Git 可执行文件、Git 库或子进程隐式读取、验证或修改仓库；commit hash 等 Git 信息只作为调用方声明处理。

## 验收标准

- Codex 可以只通过短生命周期 CLI 完成一次完整 PairFlow 协作流程，无需直接拼装 MCP/HTTP 请求。
- CLI 进程退出后，后续命令能从仓库外的用户级存储恢复正确会话。
- `wait` 遇到单次请求超时后可以继续等待，且不会累积重复连接或重复领取行动权。
- token 不写入仓库，也不在正常输出与错误输出中泄露。
- 自动化测试能证明 SDK/CLI 的协作主流程不会启动 Git 命令或 Git 子进程。
- SDK 与 CLI 的主流程、错误、取消和重试行为均有自动化测试。
- 兼容版本可正常协作；不兼容版本或缺少必需能力时，在执行状态变更前给出明确错误。
- 未启动任何常驻客户端进程时，完整协作流程仍可正常运行。
